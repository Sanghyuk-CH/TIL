## 1. 절차지향 vs 객체지향

절차지향은 위에서부터 아래로 흐르는것처럼 순차적인 처리가 중요시 된다.
장점으로 컴퓨터의 처리구조와 유사하여 실행속도가 빠르다.
단점으로 유지보수가 어렵고 코드의 순서가 바뀌면 동일한결과를 보장하지 않는다.

객체지향은 실제 세계를 모델링하여 소프트웨어를 개발하는방법
데이터와 절차를 하나로 생각하여 프로그래밍한다.
캡슐화 - 데이터와 알고리즘이 하나의 묶음으로 되어있어 내부 데이터를 감추고 외부와 상호작용하는것으로 라이브러리처럼 사용 가능하다.
상속 - 이미 작성된 클래스를 이어받아 새로운 클래스를 생성하는 기법으로 기존코드를 재활용 할수있다.
다형성 - 개념적으로 동일한 작업을 수행하는 여러 상황에 하나의 함수로 대처하는것

절차지향은 데이터를 중심으로 구현
객체지향은 기능을 중심으로 구현한다.

c언어에서 캡슐화를 쓸수 없는 이유 - 포인터를 통해서 직접적으로 메모리에 접근이 가능해서 못한다… 라고 하는데 이건 더 찾아봐야겠음

c언어는 캡슐화를 할때 구조체를 사용하는데 멤버의 가시성을 설정하는 접근 지정자 문법이 없다고 한다.
그래서 논리적 버그가 잘 일어나므로 잘 사용하지 않는다.

## 2. 데이터베이스의 트랜젝션이란

트랜젝션은 데이터베이스의 상태를 변환시키는 프로그램의 작업단위입니다.
SQL 표준에서 제일 첫번째 sql문이 자동으로 트렌젝션을 시작하고 
커밋이 호출되면 트랜젝션이 종료 되고 롤백이 호출되면 트랜젝션을 취소시키며 이전상태로 되돌아갑니다.
트랜젝션의 성질로 원자성, 지속성, 독립성, 일관성이 있으며

원자성은 데이터베이스에 트랜젝션이 모두 반영되거나 아얘 반영되지 않아야 한다는것,
지속성은 트랜젝션이 성공적으로 이루어 졌을시 결과가 영구적으로 적용되어야 하는것,
독립성은 둘 이상의 트랜젝션이 실행되고 있을시 각각의 트랜젝션이 서로에게 끼어들수 없다는 것,
일관성은 트랜젝션이 성공적으로 완료하면 언제나 일관성 있는 데이터베이스 상태로 유지하는것 입니다.
